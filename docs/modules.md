# Skooly — Modules indispensables pour faire tourner une université

## Règle de base (très importante)

Tu ne refais **PAS Gmail**. Tu ne refais **PAS Proton**.

Une université n’a pas besoin d’un nouveau fournisseur mail.
Elle a besoin :

* de **communication traçable**
* de **messages officiels**
* de **notifications fiables**

Le reste est du bruit.

---

## 1. Module Communication (pas Mail Provider)

### Ce que fait le module

* Envoi de mails officiels
* Envoi SMS / WhatsApp
* Historique des messages
* Preuve d’envoi

### Ce que le module NE fait PAS

* Boîte mail
* IMAP / POP
* Gestion de dossiers

### Intégrations

* Gmail (Google Workspace)
* Microsoft 365
* Proton Mail (SMTP)

Skooly agit comme un **hub**, pas comme une messagerie.

---

## Conclusion

Un ERP universitaire **minimal mais complet**, c’est :

1. Identités
2. Académique
3. Enseignement
4. Présences
5. Notes
6. Délibérations
7. Paiements
8. Documents
9. Communication
10. Audit

Tout le reste est optionnel.

Si tu construis ces 10-là proprement,
tu peux gérer **n’importe quelle université**.
